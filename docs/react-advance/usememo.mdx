# Introducing useMemo

> [useMemo - React 官方文件](https://zh-hant.reactjs.org/docs/hooks-reference.html#usememo)

## ⚓ useMemo Hook

```jsx
// 回傳一個 memoized 的值

const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
```

可以記憶任何 Type 的資料。

通常用在重新渲染時，不希望某些「複雜計算」再次執行以優化效能。

```jsx
// 使用 useMemo 來記憶陣列
const sortedList = useMemo(() => {
  console.log('Items sorted');
  // Return what you want to store
  return items.sort((a, b) => a - b);
}, [items]);

// empty dependency => never change
const items = useMemo(() => {
  return [5, 3, 1, 10, 9];
}, []);
```

範例中，useMemo 雖然可以記憶 Array，但同時也佔用了內存，因此只適合用於非常需要性能運算的資料上面。

## useCallback 與 useMemo 的差異

> [useCallback、useMemo 分析 & 差別](https://juejin.cn/post/6844904001998176263)

結論：`useCallback` 和 `useMemo` 都可以緩存函式的 Reference 或是緩存值，但是從更細的使用角度來說，我們偏好使用 `useCallback` 緩存函式的引用，而使用 `useMemo` 緩存計算資料的值。
